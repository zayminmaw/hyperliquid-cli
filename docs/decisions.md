# Key Decisions & Why

The load-bearing choices, with what was on the table and why this won. `PLAN.md` is
the source of truth; this records decisions made (and refined) during the build.

| Decision | Alternatives | Why this |
|----------|--------------|----------|
| **LLM owns judgment; code owns mechanics + safety.** The LLM *is* the executor, but only inside a box the gate draws. | Keep the LLM out of the order path entirely (the older ThirdEye executor's rule). | We want the LLM's judgment on *which* setup and *when* — but never on *how much* or *whether it's safe*. The gate, not the model, is the safety authority. |
| **LLM output is an input to the gate, never a bypass.** Validated + clamped before anything reaches the exchange. Schema-invalid → dropped + tallied, **never guessed**. | Trust structured output; retry/repair a bad payload. | A guessed-at decision is an unaudited trade. Dropping is safe and observable; the tally feeds the tuners. |
| **Out-of-range conviction is clamped; bad enum / missing / non-numeric is dropped.** | Clamp everything, or drop everything. | Conviction is a magnitude — clamping into `[0,1]` is well-defined and harmless. A bad *action* or *timing* enum has no safe coercion, so it's dropped. |
| **Hard caps in `.env` (off-limits to the LLM/tuner); tunable surface in `active_config.json`, clamped on load.** | One config file; let the tuner edit caps too. | This split is what makes self-tuning safe. A tunable value is always clamped against the hard caps before it can size an order — a bad/maliciously-tuned value can't reach the exchange. |
| **`paper` is the default network everywhere; `mainnet` is gated, not blocked.** | Default to testnet; block mainnet behind a build flag. | Paper needs no keys and runs the full pipeline against live marks — the right default for dev + tests. Mainnet stays reachable but requires `HL_ENABLE_MAINNET=1` **and** `--network mainnet` **and** a typed confirm. |
| **`anthropic` + live-exchange deps are lazy-imported.** | Top-level imports; require all deps always. | Paper mode and the entire test suite must run with **no keys or signing libs present**. Lazy imports (`core/llm.make_client`, the SDK inside `hyperliquid.py`) keep the hot import path clean. Verified by a no-extras venv. |
| **Reads go over httpx `/info`, not the SDK `Info`.** | Use the SDK for everything. | Keeps marks/book/positions keyless and SDK-free so paper stays dependency-light. The SDK + `eth_account` are write-only (signing). Don't "simplify" reads onto the SDK — it breaks the keyless invariant. |
| **Order-path model = `claude-sonnet-5`; daily tuner = `claude-sonnet-5`.** | Opus for the tuner; one large model for both; a smaller/cheaper model (e.g. Haiku) for the hot loop. | The gate is the real safety authority, so the hot loop needn't be the largest model — Sonnet 5 already outperforms the older Sonnet 4.6 default at equal-or-lower cost, whereas Haiku's fit for this judgment-heavy, low-temperature verdict is unproven; validate any move to a smaller model via `exec shadow` before trusting it. The tuner runs once a day, out of path — Sonnet 5 reaches near-Opus quality there for a fraction of the cost, and every proposal is human-reviewed before `promote` regardless of which model wrote it. |
| **Idempotency key recorded *before* the order is placed.** | Record after a confirmed fill. | A crash between placing and confirming must skip (a missed trade), never double-fire. The key is released only on a *definitive* reject; transport errors keep the key and re-raise. |
| **Executor entry is a MARKET order.** | A resting GTC limit at the candidate's entry. | A resting limit may never fill, leaving the ledger and protective triggers tracking a phantom position. A MARKET entry means *accepted ⇒ filled*; the runner then reconciles the trade + sizes protection from `OrderResult.filled_size`/`avg_price`, not the intended order. (Pre-mainnet review finding H1.) |
| **Candidate side is inferred from level geometry** (`sl<entry<tp` = long). | Require an explicit side field. | The levels already encode direction; an explicit side that contradicts them is a bug surface. Incoherent levels are rejected at intake *and* at the gate. |
| **Native exchange-side SL/TP is a hard mainnet prerequisite** (§13 Q6, user-confirmed). Scoped to testnet+mainnet. | Manage SL/TP in-process; mainnet-only triggers. | A crashed executor must not leave a position unprotected. A live entry that can't be protected is emergency market-closed and its ledger row resolved as `aborted` (ledger-first: the row is written on fill, *before* protection, so a crash mid-protection still leaves a managed position; the key is already spent so no re-fire). Whichever trigger did place is cancelled. Scoping to testnet too exercises the safety path before real money. |
| **Self-tuning is out-of-path and propose→approve** (§13 Q4, default kept). | Auto-apply tuned values. | A human approves before anything goes live, never auto-applied to mainnet. Cohorts are sample-gated (`<5` ⇒ no model call); config is clamped on propose, on promote, and on load. |
| **Resolution is executor-side** (write trade outcomes from SL/TP/expiry). | Defer resolution / rely on exchange fills only. | The tuners need resolved-outcome cohorts and the graduation verdict needs realized R-multiples — both require a trades ledger the executor maintains each pass. |
| **Alerts = JSONL + stderr, no deps/keys.** | A webhook/email channel. | Keeps the safety path dependency-free and keyless. A webhook tailing `alerts-<network>.log` is a clean optional follow-up. `None` alerter in shadow/tests = silent. |
| **Halt alert is edge-triggered.** | Alert every pass while halted. | A tripped breaker/loss-limit shouldn't spam the run loop. The alert fires on a *change* of halt state (tracked in `meta`). (Review finding L5.) |
| **Regime is computed in code (Kaufman ER), not by the LLM.** Fed to the gate + a 12-bar OHLC tail handed to the model. | Ask the LLM to label the regime; skip regime entirely. | Regime gates *whether* a setup is allowed — a safety-relevant judgment, so it must be deterministic and auditable, not a model opinion. The candle fetch is keyless/best-effort (15m×48, once per coin per pass); a feed hiccup degrades to `None` and the gate skips the check rather than fabricating a signal. |
| **An `act + wait` decision is deferred, not rejected.** Parked in a `deferred` table and re-checked with fresh data, intercepted *before* the gate. | Reject WAIT outright (lose the setup); let the gate handle WAIT. | A good setup that's simply early shouldn't be thrown away. Intercepting before the gate keeps the gate a pure act-now decision. Re-checks stay *within* the freshness window, are capped (`HL_FOLLOWUP_MAX_ATTEMPTS`), use fresh enrich/candles/regime each time, and freeze while the breaker is tripped — so deferral can never outlive a candidate's validity or bypass the kill switch. |
| **Prompt payloads label their data; the prompts state the mechanics the model lives under.** Candles carry `interval`/`order`; recent rows carry `coin` + `minutes_ago` (both lists newest-first); the decision prompt explains the action×timing combinations, that conviction below `min_conviction` sizes to zero, and asks for a 2-4-sentence rationale-first (the only reasoning space under a forced tool call — per Anthropic's tool-choice guidance); config-tool field descriptions state units + clamp bounds (strict mode can't encode numeric ranges). | Bare data dumps; keep prompts terse and let the model infer. | A model can't infer a candle timeframe, the recency of an unlabeled row, or an unstated clamp — it either guesses or wastes its verdict on values the code will silently clamp. Research also shows verbalized confidence clusters high, so the prompt anchors the conviction scale and warns against clustering. Balanced act/skip framing prevents literal-instruction skip-bias from starving the shadow book (and therefore the sample-gated tuners) of outcomes. |
| **No mark ⇒ reject before the LLM call.** | Let the gate's mark-sanity check catch it after the decision. | Same outcome either way (the gate would reject), but the paid decision call is skipped. Logged as a rejection with an explicit reason so the audit trail stays complete. |
| **`stop_reason` rides on every `DecisionResult`.** | Log only the drop note. | A drop caused by truncation (`max_tokens`) or a refusal is a different operational problem than malformed output — the audit log needs to distinguish them to debug drop rates. |
| **Sentry mechanics are deterministic code, not LLM (Phase 6a first).** Breakeven ratchet, ATR/percent trail, and the scale-out ladder run as clamped tunable rules (all default off); the LLM manager (6b+) will be judged *against* this baseline in shadow before it may act. | Give the LLM the trade-management wheel immediately. | Practitioner trade management is pre-committed rules, and live LLM-trading experiments (Alpha Arena, on this very exchange) lost to indiscipline, not bad prediction. The engine also gives 6b a measurable value-add benchmark. (PLAN.md §14.) |
| **R math anchors to `initial_sl`; a stop-out on the profit side of entry books `won`; a `scaled` partial counts as a win.** | Keep measuring risk off the working `sl` and call every stop-hit `lost`. | Once the stop ratchets, `\|entry − sl\|` shrinks (inflating R), and a trailed exit in profit is profit taken on purpose — booking it `lost` would poison the win-rate cohorts and the graduation expectancy that gates mainnet. |
| **Sentry never lowers protection.** The ratchet is engine-enforced; a live stop moves place-new-then-cancel-old; a rejected replacement keeps the old level on both exchange and ledger; scale-outs are idempotent (`sentry:scale:<id>` recorded before the order). | Cancel-then-place; trust the rules to only propose tighter stops. | The position must never be naked, the ledger must never claim protection tighter than what the exchange holds, and a crash between order and ledger split must not close the fraction twice — same discipline as `fire()`. |
| **The LLM manager (6b) is shadow-only and judged against the rule baseline.** Every proposal is logged *paired* with what the 6a engine would do at the same instant, before the rules mutate the book; the baseline is never shown to the model. Menu is bounded (hold default; tighten/reduce/close/extend_tp; **no ADD until 6d**); an action whose own parameter is unusable is dropped, never defaulted. | Let the LLM manage live immediately; include the baseline in its context. | The whole 6c go/no-go question is "does judgment beat the rules?" — that needs an untainted paired log, not an LLM that anchors on the rule's answer. Structural-only validation keeps the validator honest: direction sanity (does a tighten tighten?) belongs to the 6c management gate, one place, not two. |
| **6c live actions pass a management gate whose churn state IS the sentry log.** Eval spacing, per-position cooldown + daily budget, the global LLM-call budget, and the extend↔bank opposing window are all computed from logged rows (`HL_SENTRY_*` hard caps in `.env`), never from in-memory counters. Breaker/loss-limit ⇒ only risk-reducing actions pass. Mainnet refused until graduation (6d). | Track churn in process state; trust the prompt's anti-churn instructions. | Alpha Arena's losers died of churn, so the throttles must be code, must survive a restart, and must be off-limits to the model and the tuner — the log is the one place a restart can't wipe. |
| **A judgment CLOSE books won/lost by the sign of realized P&L; one partial per trade (ladder or LLM); a `scaled` row is a win only when its realized is positive.** | Book LLM closes as `closed`; let reductions stack; keep counting every `scaled` as a win. | A deliberate exit resolves the entry decision, so its outcome must carry sign into the cohorts; stacked partials are churn with a worse crash story (the shared `sentry:scale:<id>` key + `scaled_out` flag keep 6a's idempotency guarantees); and the 6c manager can bank a partial *loss*, which must not flatter win-rate. |
| **ADD is nominate-and-size, applied raise-stop-first.** The model may only nominate a pyramid with a raised stop; the gate admits it solely for winners ≥ `sentry_add_min_r` and the code computes the size — min(unrealized-profit coverage, ½ the coin's total size, notional room, leverage room on the coin's TOTAL size incl. sibling rows) — under a lifetime per-coin add budget. Apply raises the whole position's stop before any new size exists (refused raise ⇒ no add); the slice becomes its own ledger row (entry at fill, `initial_sl` at the raised stop) with its own native protection, emergency-closed + `aborted` on protection failure; the idempotency key is the coin's add ordinal. | Let the model pick the add size; blend the add into the original row; add first and raise after. | Every pyramid rule from practice (winners only, smaller adds, risk covered by open profit, stop up with every add) becomes code, not prompt. A separate slice row keeps R attribution and tuner data honest where blending would corrupt both. Raise-first means a crash mid-add always leaves LESS risk than before, never more. |
| **Mainnet management is earned on the testnet book.** `graduation_for_management` runs `safety/graduation.assess` over TESTNET resolved trades (real + shadow); `hl sentry manage`/`run --manage` on mainnet refuse, naming the failing checks, until it clears. | Gate on the mainnet book itself, or ship 6d ungated. | §7's principle: privileges over real money are earned on fake money — the mainnet book is tiny by construction, so judging on it would either block forever or prove nothing. |
| **The live manager sees only what actually happened (`prior_actions` excludes shadow rows), and manages only the real book.** | Include shadow proposals as history; manage the hypothetical book too. | Observed live: the model read its own unapplied shadow proposal as "a prior action that moved the stop". And the shadow book must stay a clean record of *entry* decision quality — LLM-managing it would conflate the two experiments. |
| **`hl sentry once|run` is a watch pass: `run_once(include_intake=False)`.** Sentry re-checks due WAIT deferrals (and may enter them through the normal decision + entry gate) and manages/resolves the book on its own cadence, but never consumes the intake stream. | A separate deferred-recheck code path for sentry. | One shared implementation means the deferred table, follow-up attempt counters, idempotency keys, and breaker freeze behave identically no matter which loop gets there first — running `exec run` and `sentry run` side by side cannot double-spend an attempt or double-fire. |
| **`hl config set/edit` give direct operator control of the tunable surface; the clamp is the safety boundary, not the edit mechanism.** `set` refuses hard-cap keys (those are `.env`), coerces the value to the field's type, and writes the *clamped* result; `edit` re-validates + clamps on save. Both write byte-identical files to a `tune promote`. | Keep them stubbed and force every tunable change through `hl tune`. | The tuner needs a ≥5-sample cohort, so it can't set anything on a fresh install; and the split exists precisely so *any* value reaching the surface is clamped on write and again on load — so a manual setter is exactly as safe as a promotion, and the advertised verb should work. |

## Deviations from the original plan (recorded)

- **Reads use httpx, not the SDK `Info`** — so paper stays keyless; `httpx` moved into core deps. (Action item 1.4.)
- **Watch modes are poll-based `rich.Live`**, not native websockets — call sites are unchanged so an `Info.subscribe` upgrade is a later refinement. (Action item 1.9.)
- **Regime is a single coarse signal** (Kaufman efficiency-ratio, 15m×48) rather than a multi-timeframe model — deliberately simple for now; the gate skips its regime check when the feed yields too few bars (`None`) rather than fabricating a signal. (Action items 3.1, 3.7.)
- **`equity()` is unified-account aware** — Hyperliquid shipped unified accounts (now the testnet default), where the perp `clearinghouseState.accountValue` is only committed position margin (~0 when flat) and the tradeable collateral lives in the spot clearinghouse. The backend detects `userAbstraction=="unifiedAccount"` (cached) and reads spot USDC + open-position uPnL for those accounts; standard accounts keep the `accountValue` read. Without this, testnet (and any unified account) reads equity ~0 → sizing and the equity-based caps break. Found on the first real testnet drill (2026-07-19).
- **Externally-closed positions book the real exit fill (wave-2 L)** — a vanished position (native-trigger, manual flatten, liquidation) was booked at the mark; it is now priced at the actual closing fill via `Exchange.recent_fills` (live maps `user_fills_by_time`; paper `[]`), falling back to the mark when no fill matches, skipped for scaled trades. `Fill` field semantics (`dir`, gross `closedPnl`, separate `fee`) are verified against a live testnet fill, not assumed.
- **Liquidation-proximity guard (wave-2 M)** — an open position whose mark sits within `HL_MIN_LIQUIDATION_DISTANCE_PCT` (default 5%) of its exchange `liquidationPx` raises an edge-triggered critical `liquidation_near` alert, and `exec status` shows `liq_dist%`. `Position.liquidation_px` is null-safe: verified live that Hyperliquid returns `liquidationPx: null` for a well-collateralised cross position, so None means "no near risk" (never fail-closed on it). Scope: the in-trade alert; a pre-fire maintenance-margin gate (`crossMaintenanceMarginUsed`) is the tracked follow-on.
- **Realized P&L and R are net of the taker fee (wave-2 K)** — hl-cli entries and closes are IOC takers, so `resolve._pnl` subtracts a round-trip taker fee (`rmath.taker_fee`, `HL_TAKER_FEE_PCT` default 0.045) and records it in `fee_paid`; graduation and the tuner therefore promote on cost-honest expectancy. Paper models the fee (real per-fill `fee` reads for live, and funding, are the tracked follow-on — funding's sign was deliberately not shipped unverified). The chosen semantics (net, not gross-plus-a-report-column) mean graduation thresholds are net expectancy. `taker_fee_pct=0` (the test caps) reproduces the pre-K gross numbers exactly. **Scope:** the resolve path only for now — sentry-close and abort fee-netting are the immediate follow-on (both off/excluded in the default config, so all default-config evidence is already net).
